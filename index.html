Impossible de compiler les guest additions et les drivers vidéos de virtualbox sous Gentoo

Rédigé par jeremy - 19 septembre 2015 - Aucun commentaire

Je voulais me faire une VM sous gentoo quand je m’aperçois à la fin de l’installation que, ni les drivers vidéos, ni les Guest Additions ne sont possible à compiler.

On remarque des erreurs de ce type :

Il semble qu’il manque une fonction … après recherche j’ai trouvé que cette erreur était due à l’utilisation de GCC-4.8.

Il faut donc passer à GCC-4.9, je regarde donc quelle version est disponible grâce à eix :

Je remarque que la version 4.8.2 est bloquer par Keyword, je déblock donc cette version :

Apres une longue attente, gcc devrais etre installé reste à selectionner le nouveau compilateur :

On liste les versions de GCC installé

On sélectionne la bonne version

En listant une nouvelle fois, on appertisation que le bon profile est sélectionné.

Nous pouvons maintenant installer les Guest additions de virtualbox !

Pour supprimer la version innutiliser de GCC, je vous avous que je prefere recompiler tout le système, je ne sais pas si c’est obligatioire mais c’est fortement recommander avant de lancer la commande de néttoyage des dépandances.

Apres une (treeees) longue attente le système est entierement recompilé, on peut donc nettoyer les dépendances orpheline.
